Actions of programs in org.jsresources package

These classes were adapted from Java Sound Resources with the software repository being named jsresources at SourceForge.net

The software makes use of the Tritonus: Open Source Java Sound project. I used a version of the jar file from the Maven repository.

DumpJSInfo

  • java org.jsresources.DumpJSInfo

    This class prints information about the various devices that are part of the MIDI system.

DumpSequence

  • java org.jsresources.DumpSequence /Users/bradleyross/Downloads/AGNICRT.MID

    This dumped the contents of the MIDI file in a human readable format.

MidiFileInfo

  • java org.jsresources.MidiFileInfo /Users/bradleyross/Downloads/AGNICRT.MID

    This displays information about the MIDI file.

MidiInDump

  • java org.jsresources.MidiInDump -l

    When I ran this command, I had a KORG nanokey controller attached. This MIDI device had the name KEYBOARD and was displayed as input device 1. Input device 3 was the Real Time Sequencer.

  • java org.jsresources.MidiInDump -d KEYBOARD

    While the program was running, a line of text was displayed whenever a key on the KEYBOARD device was depressed or released. This continued until the program was halted with CTRL-C.

  • java.org.jsresouces.MidiInDump -n 1

    This had the same results as above since the KEYBOARD device was device number 1.

MidiNote

  • java org.jsresources.MidiNote "Gervill" 38 60 10000

    Gervill was the name of the Synthesizer. When I entered the command, something appeared to happen, but there was no sound. It may be necessary to initialize the synthesizer. The logical approach would be to compare the code MidiNote to MidiPlayer.

MidiPlayer

  • java org.jsresources.MidiPlayer /Users/bradleyross/Downloads/AGNICRT.MID

    When the command is entered, the MIDI file is played through the speakers.
